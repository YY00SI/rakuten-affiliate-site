## 6月分析への問題提起・引継ぎ事項

次回の分析を担当するAIエージェントは、以下の論点およびフロー上の制約を考慮して作業を行ってください。

### 1. ネットデータ収集フローの制約（最重要）
- **状況**: 現在のAI環境には直接のヘッドフルブラウザ自動操作（Playwright/Selenium等のアタッチ）が標準搭載されておらず、単なる `Start-Process` で開いたブラウザは操作できません。
- **対応策**: 
  1. ユーザーに各ダッシュボード（GSC, GA4, 楽天アフィリエイト）から必要な数値をコピー＆ペーストしてチャットに共有してもらう（手動フロー）。
  2. あるいは、自動取得を試みる場合は、事前にデバッグポート（`--remote-debugging-port=9222`）を有効にしてChromeを起動し、そこにアタッチするスクリプトを書いて実行する。

### 2. GSCインデックス登録状況の検証
- **問題提起**: 現在全61記事中、GSCやGA4で反応が確認できているのは6記事のみ（約9割が反応なし）。これが**「インデックス未登録」なのか「登録済みだが表示0」なのか**の切り分けができていません。
- **アクション**: GSCのURL検査を使用し、優先ページ（トップ、ゲーミングPC、ウルトラワイドモニター等）のインデックス状態を網羅的に確認し、未登録の場合はインデックス登録をリクエストしてください。

### 3. CTA（価格を見る）ボタンの導入効果検証
- **問題提起**: GSCクリック8に対して楽天アフィリエイトへの遷移（クリック）は1のみに留まっており、遷移率（CTR）が低いです。比較表下に追加した「価格を見る」ボタンおよびGA4イベント（`rakuten_affiliate_click`）の受信状況を確認し、効果があったかを検証してください。

### 4. 勝ち筋カテゴリ（ポータブルゲーミングPC/UMPC）の需要検証と拡充
- **問題提起**: UMPC系で検索露出とクリックが出ているため、このトレンドを逃さずに関連するロングテール記事を追加する必要があります。6月中に該当カテゴリの競合ベンチマークとキーワード需要を調査し、コンテンツを追加してください。
