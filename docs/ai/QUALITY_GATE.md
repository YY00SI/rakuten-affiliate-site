# Quality Gate — 軽量モデルでも品質を落とさない仕組み

本ドキュメントは、Gemini Flash 級の軽量モデルを含む**すべての AI** が  
`LifeTech Select` 水準の記事を安定して生産するための品質保証フレームワークです。

---

## 0. 設計思想

| 原則 | 説明 |
|:---|:---|
| **入力で縛る** | モデルの推論力に頼らず、テンプレートと制約で出力を固定する |
| **機械で検証する** | 人間の目視判断を最小化し、バリデータとチェックリストで合否を決める |
| **ロールバック可能** | 公開後に問題が見つかっても、YAML を戻すだけで即座に復旧できる |

## 2. 軽量モデル向けプロンプト設計原則

軽量モデル（Gemini Flash、GPT-4o-mini 等）は以下の弱点を持つ。  
品質ゲートはこれらを**構造で回避**する。

| 弱点 | 対策 |
|:---|:---|
| 長い指示を途中で忘れる | 指示は箇条書き＋チェックリスト形式。1回のプロンプトに1記事分だけ |
| 抽象的な「良い文章を書け」に応えられない | すべてのフィールドに**具体的な文字数制限**と**禁止語**を設定 |
| 既存記事と重複する内容を生成する | 重複防止チェックリスト（§5）を渡し、既存 slug/h1/title を列挙する |
| 「すごい」「高性能」等の空疎な表現を使う | `discouraged_terms` を明示し、セルフチェックで排除させる |
| 楽天キーワードに汎用語を使う | 「ブランド名+型番」のみ許可する原則をプロンプトに含める |
| scores の key を test_criteria と不一致にする | テンプレート上で必ず scores の key と criteria の id を対応させる |

### 2-1. 1記事単位プロンプトテンプレート

軽量モデルへ渡すプロンプトは以下の構造に統一する。

```
## 指示
あなたは LifeTech Select のシニア編集者です。
以下の入力情報に基づき、config/article_blueprint.template.yaml の
全フィールドを埋めてください。出力は YAML のみ。前置きと言い訳は禁止。

## 入力
- カテゴリ: {category}
- テーマ: {theme}
- 検索意図: {search_intent}
- 想定読者: {target_reader}
- 候補機種: {brand_model_list}
- 最低価格: {min_price}
- 評価軸: {test_criteria_list}
- 既存記事の slug 一覧: {existing_slugs}

## 制約
1. intro は 80〜220 文字。読者の悩みと記事の約束を書く。
2. analysis_insight は 70〜220 文字。後悔ポイントを言い切る。
3. meta_description は 70〜140 文字。
4. title は 28〜70 文字。年号・厳選数・ベネフィットを含む。
5. test_criteria は 2〜4 個。抽象語（満足度、人気）禁止。
6. products_extra の各商品で best_for, scores, analysis_why,
   pros (2〜5個), critical_cons, maintenance_reality,
   cost_performance を必ず埋める。
7. scores の key は test_criteria の id と完全一致させる。
8. rakuten_params.keyword は「ブランド名 型番」のカンマ区切り。
9. slug は既存記事と重複禁止。
10. 「すごい」「高性能」「コスパが良い」等の空疎な形容は禁止。

## 出力
YAML ブロックのみ。コードフェンスで囲む。
```

## 3. セルフチェックリスト（QUALITY_CHECKLIST）

AIが記事を出力した後、**自分自身で**以下を全項目チェックする。  
1つでも NG があれば修正してから提出する。

### 3-1. 構造チェック

- [ ] `id` / `slug` / `h1` / `title` が既存記事と重複していない
- [ ] `release_date` が YYYY-MM-DD 形式で、未来日または当日
- [ ] `category_id` が `beauty` / `home` / `trend` / `work` のいずれか
- [ ] `type` が `daily` または `sns`
- [ ] `test_criteria` が 2〜4 個
- [ ] `products_extra` が 1〜5 個
- [ ] 各 `products_extra` に 7 必須フィールドがすべてある
- [ ] `scores` の key が `test_criteria` の `id` と完全一致
- [ ] `pros` が 2〜5 個
- [ ] `rakuten_params.keyword` が空でない
- [ ] `rakuten_params.hits` が 1〜30 の整数

### 3-2. 文字数チェック

- [ ] `title`: 28〜70 文字
- [ ] `meta_description`: 70〜140 文字
- [ ] `intro`: 80〜220 文字
- [ ] `analysis_insight`: 70〜220 文字

### 3-3. 内容品質チェック

- [ ] `intro` に読者の悩みと記事の約束が書かれている
- [ ] `analysis_insight` に「後悔ポイント」が明確に言い切られている
- [ ] 各 `best_for` が生活シーンで具体的に書かれている
- [ ] 各 `critical_cons` が購入判断に効く現実（薄い欠点ではない）
- [ ] 各 `cost_performance` に価格の高さを回収するロジックがある
- [ ] 各 `maintenance_reality` に手入れの手間が具体的に書かれている
- [ ] `rakuten_params.keyword` に汎用語（おすすめ/人気/ランキング）が含まれていない
- [ ] `forbidden_words` に基本禁止語（中古/訳あり/ジャンク/ふるさと納税/返礼品/部品/パーツ/ケース/カバー/フィルム/スタンド）が含まれている

### 3-4. 禁止表現チェック

以下の表現が記事内に含まれていないことを確認する。

- [ ] 「すごい」「高性能」「コスパが良い」等の空疎な形容
- [ ] メーカー説明文のコピー
- [ ] 「おすすめ」「人気」だけの汎用的なキーワード
- [ ] 商品本体以外のアクセサリや周辺部品の混在
- [ ] 複数の検索意図の混在

## 4. 機械検証の合格基準

| 検証ステップ | コマンド | 合格条件 |
|:---|:---|:---|
| **構造検証** | `python src/validate_articles.py` | `errors=0` 必須 |
| **商品取得** | `python src/fetch_products.py` | 取得件数 > 0 |
| **サイトビルド** | `python src/build_site.py` | `[CRITICAL QA ERROR]` なし |
| **公開監査** | `python src/audit_site.py` | `errors=0` 必須 |

### 4-1. よくある検証エラーと対処

| エラー | 原因 | 対処 |
|:---|:---|:---|
| `duplicate slug` | 既存記事と slug が被っている | slug を変更 |
| `scores missing keys` | test_criteria の id と scores の key が不一致 | key を揃える |
| `test_criteria count` | 評価軸が1個以下または5個以上 | 2〜4個に調整 |
| `products_extra missing field` | 7 必須フィールドの抜け | 該当フィールドを追記 |
| `title length outside range` | title が短すぎ/長すぎ | 28〜70文字に調整 |
| `キーワード に合致する製品が見つかりません` | keyword が楽天APIの検索結果に一致しない | keyword をブランド名+型番に修正 |

## 5. 重複防止プロトコル

新規記事を作る前に、以下の情報を AI に渡すこと。

1. **既存 slug 一覧**: `articles.yaml` の全 slug を列挙
2. **既存 h1 一覧**: 全 h1 を列挙
3. **既存 title 一覧**: 全 title を列挙
4. **既存 keyword 一覧**: 全 `rakuten_params.keyword` を列挙

→ これにより、テーマの重複・URL の衝突・キーワードの共食いを防ぐ。

## 6. 品質レベル定義

| レベル | 定義 | 判定基準 |
|:---|:---|:---|
| **A（公開可）** | すべてのチェック項目が OK | errors=0, warnings=0, セルフチェック全通過 |
| **B（要修正）** | 軽微な問題あり | errors=0, warnings ≤ 3, セルフチェック1〜2個 NG |
| **C（再作成）** | 重大な問題あり | errors > 0, またはセルフチェック3個以上 NG |

- **A** → そのままマージ可能
- **B** → 指摘箇所を修正後、再検証してマージ
- **C** → テンプレートから作り直し

1. **検証で頻出するエラー** → チェックリストに追加
