# 2026-05-24 収益化・根拠強化アップデート

## 背景
現状の課題は、記事内に独自分析の表現はある一方で、商品ごとの判断根拠がページ上で検証可能な形になっていなかったこと。特に「ここが支持されている」「不都合な真実」「メンテナンスの現実」「コスパの正体」が、口コミ本文・仕様・外部情報のどこから来た判断なのかが曖昧だった。

## 実施内容
- 楽天APIの商品取得を、固定の指名買い候補だけでなく市場探索キーワードも併用する実装へ拡張。
- 楽天市場商品検索APIのエンドポイントを現行版 `2026-04-01` へ更新。
- 各商品JSONに `candidate_summary` と `hidden_gem_candidates` を保存するよう変更。
- メインランキングとは別に「口コミ評価から拾った掘り出し物候補」を表示する枠を追加。
- 各商品カードへ「根拠メモ」を追加し、レビュー評価・件数、商品説明内の仕様シグナル、商品説明抜粋を表示。
- 各ページ冒頭の「独自インサイトレポート」に、掲載モデル数、確認したレビュー母数、加重平均評価、価格帯、根拠スコープを表示。
- 口コミ本文を取得していない状態で、本文を読んだかのような断定をしないルールを `CONTENT_CONTRACT.md` と `QUALITY_GATE.md` に追加。
- 既存の構造エラーを修正し、`validate_articles.py` と `audit_site.py` を `errors=0, warnings=0` にした。

## 判断
楽天市場商品検索APIで取得できる主な根拠は、商品名、価格、商品説明、商品URL、アフィリエイトURL、レビュー件数、レビュー平均などであり、レビュー本文そのものではない。したがって、現時点の正しい表現は「レビュー本文を解析した」ではなく、「公開レビュー評価・件数・商品説明・仕様表記を横断して判断した」。

公式仕様上、楽天APIは `itemCaption`, `reviewCount`, `reviewAverage`, `affiliateUrl` などを返せるが、レビュー本文フィールドは商品検索APIの出力項目として扱えない。今後、口コミ本文単位の分析を行う場合は、別の取得経路・許諾・引用ルールを設計する必要がある。

## 残る課題
- Web上のブログ記事・専門レビュー記事の本文収集は、今回の自動取得パイプラインにはまだ入れていない。
- 外部記事を根拠化する場合は、検索結果の取得、出典URLの保存、要約、引用制限、更新日確認を別パイプラインとして実装する必要がある。
- 収益化は検索インデックス、検索順位、CTR、楽天側CVRに依存するため保証はできない。ただし、今回の変更で「根拠が曖昧なアフィリエイト記事」から「判断根拠を表示する比較記事」へ改善した。
