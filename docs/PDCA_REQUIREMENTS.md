# PDCA Requirements

`rakuten-affiliate-site` の収益化PDCAに必要な情報と運用項目です。パスワードは保存しない。

## 目的

- アクセスゼロ、収益ゼロの状態から、検索露出、クリック、成果のどこで詰まっているかを毎回切り分ける。
- 記事追加だけで終わらせず、検索クエリと成果データから次の記事テーマを決める。

## 必要サイト

| 用途 | サイト | 必要情報 |
| --- | --- | --- |
| 成果確認 | 楽天アフィリエイト | ログイン可否、クリック数、成果件数、売上、報酬、商品別データ |
| 商品取得 | 楽天ウェブサービス | `RAKUTEN_APP_ID`, `RAKUTEN_AFFILIATE_ID` |
| 検索流入 | Google Search Console | 対象プロパティ、検索クエリ、ページ、CTR、平均掲載順位、インデックス状況 |
| 行動分析 | GA4 | PV、参照元、滞在、クリックイベント |
| 公開確認 | GitHub / GitHub Pages | リポジトリ、Pages URL、デプロイ状況 |

## ID/PW管理

- パスワードはこのリポジトリに保存しない。
- IDも原則としてドキュメントには残さず、必要時にユーザーから再提示またはブラウザ/パスワード管理ツールで入力する。
- `.env` に保存してよいのはAPIキーなどの実行用シークレットのみ。Git管理対象にしない。
- PDCA実行時は、取得した数値だけを記録し、ログイン認証情報は記録しない。

## 取得導線

### 楽天アフィリエイト

- `レポート > サマリ`
  - 取得対象: `クリック数`、`成果件数`、`売上`、`報酬`
- `レポート > 注文明細`
  - 取得対象: `成果商品`
  - `対象データが見つかりませんでした` の場合は `成果商品: なし`
- `レポート > ショップ別`
  - 取得対象: `ショップ別クリック`
  - 現行UIで確認できた最小粒度がショップ別であり、商品単位へ降りられない場合は `商品別クリック: 未取得`
  - ショップ別クリックは補足データとして記録してよい
- 対象期間
  - 楽天は画面上の期間表現をそのまま記録する
  - 例: `直近30日`、`2026年06月`

### Google Search Console

- 対象プロパティは `https://yy00si.github.io/rakuten-affiliate-site/`
- `検索パフォーマンス`
  - 取得対象: `合計クリック`、`合計表示`、`平均CTR`、`平均掲載順位`、`上位クエリ`、`上位ページ`
- `ページのインデックス登録`
  - 取得対象: `登録済みページ数`、`未登録ページ数`、`未インデックス理由`
  - URL一覧を十分に取得できない場合でも、件数と理由は記録する

### GA4

- 対象プロパティは `LifeTech Select`
- `ホーム`
  - 取得対象: `PV`、`アクティブユーザー`、`主な参照元`、`イベント内訳`
- `rakuten_affiliate_click`
  - ホームで見えない場合はイベント一覧で確認する
  - 一覧に存在しないことを確認できた場合のみ `0` と記録する

### 公開URL確認

- 少なくともトップ1本、重点記事1本、監視対象記事1本を確認する
- `200 OK` を返した場合は `公開URL確認: OK`
- ネットワーク制約などで到達確認不能な場合は `未取得`

### ストック記事の最終公開予定日

- `config/articles_stock.yaml` の末尾の `release_date` を採用する

## 未取得ルール

- 導線が現行UIに存在しない、画面未発見、権限不足、Chrome接続失敗、ネットワーク制約の場合は `未取得`
- 推測で補完しない
- 代替観測値は主項目とは分けて `補足` として扱う
  - 例: `商品別クリック: 未取得`、`補足: ショップ別クリックは 4ショップで計5`

## 判断基準

- GSC表示0: インデックス、内部リンク、サイトマップ、テーマ需要を確認。
- 表示ありクリック0: タイトル、メタ、検索意図、順位を改善。
- クリックあり楽天クリック0: CTA位置、商品リンクの視認性、比較表を改善。
- 楽天クリックあり成果0: 商品選定、価格帯、在庫、購入直前の不安解消を改善。
- 成果あり: 同カテゴリの型番別・悩み別記事を追加。

## 次に必要な情報

- Google Search Console の現在値
- 楽天アフィリエイトの直近30日値
- GA4導入有無
- GitHub Pages の公開URL
- 楽天APIキーが現在の `.env` で有効か
- どの記事がインデックス済みか
