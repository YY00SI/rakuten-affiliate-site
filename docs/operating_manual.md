# 標準運営マニュアル (SOP)

本ドキュメントは、ライフテック・セレクト社の新規記事作成および日次運用の標準手順を定めるものである。

---

## 1. 新規記事の立ち上げ手順
1.  **市場調査**: 楽天市場で需要が高く、報酬単価が魅力的なカテゴリを特定。
2.  **config/articles.yaml への登録**:
    - `id`, `category_id`, `slug` を定義。
    - **【最重要】`core_kws` の定義**: `fetch_products.py` 内に、その商品の本体を指すキーワード（例：炊飯器, rice cooker）を必ず登録し、ノイズフィルタを有効にする。
3.  **解析データの定義 (任意だが推奨)**:
    - ランキング上位3位までは、`products_extra` に独自の解析コメント（メリット・デメリット）を記述し、ブランド力を高める。

## 2. コンテンツ制作の三原則（パイオニアからの吸収）
全記事、以下の3つの視点を必ず組み込み、自動生成の枠を超えた付加価値を提供する：

1.  **[Logic] 比較のインフラ (mybest流)**:
    - ユーザーが自分で選べるよう、主要製品を横並びにした比較表を冒頭に配置。
    - 独自の検証軸（速乾性、静音性等）を設け、データを構造化する。
2.  **[Emotion] 生活の解像度 (ROOMIE流)**:
    - 導入文（Intro）ではスペックを語らず、ユーザーの「不満」や「理想の生活」を語る。
    - 「Best for:」タグを用い、特定のライフスタイルへの適合性を明示する。
3.  **[Truth] 誠実な解析 (価格.com流)**:
    - 良い点だけでなく、レビューから抽出した「本音のデメリット」を必ず併記。
    - 「リサーチ結果の要約」では、価格相応の価値があるかを客観的に判定する。

## 3. CEOによる品質監査プロセス (デプロイ前チェック)
デプロイ（GitHubへのプッシュ）前に、CEOは必ず生成された `docs/` 配下のHTMLを以下のリストで抜き打ち検査する：

### 監査チェックリスト
- [ ] **不純物フィルタ**: 商品名に「ふるさと納税」「返礼品」「中古」が1件も含まれていないか。
- [ ] **テーマ一貫性**: 掃除機の記事にテーブルが混じっていないか、スピーカーの記事にスタンドが混じっていないか。
- [ ] **UI整合性**: PCで閲覧した際に、右側の「リサーチ結果」がすべて読めるか（見切れていないか）。
- [ ] **排他制御**: 手動解説がある商品に、自動生成の要約文が重複して表示されていないか。
- [ ] **リンク有効性**: すべての「楽天市場でチェック」ボタンが動作するか。

## 3. 異常発生時のエスカレーション
- 商品取得が0件になった場合、フィルタが厳しすぎる可能性があるため、`core_kws` の判定を緩和し、再取得を行う。
- 新たなノイズ（例：景品目録など）が発見された場合、直ちに `fetch_products.py` の `NEGATIVE_KEYWORDS` に追加し、全記事を再ビルドする。
